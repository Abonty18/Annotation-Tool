Review Annotation Guidelines

Section Description
Objective The goal of this annotation task is to categorize the reviews based on whether they are related to privacy features, privacy bugs, or neither.
Label Categories
  • Privacy-Related Feature Request (Label 1): Assign this label if the review is discussing a feature request of the app related to user privacy.
  • Privacy-Related Bug (Label 2): Assign this label if the review is reporting a bug or issue related to user privacy.
  • Not Privacy-Related (Label 0): Assign this label if the review is not discussing any privacy-related aspects.
Annotation Instructions
  1. Read the Review: Carefully read and understand the content of the review.
  2. Identify Privacy Context: Determine whether the review is discussing any issue related to user privacy.
  3. Assign Appropriate Label: Based on the content of the review, assign one of the three labels.
Examples Reviews like advising or requesting to update any privacy-related feature of the app should be labeled as Privacy-Related Feature Request (Label 1). If a review reports personal data exposure due to a bug, label it as Privacy-Related Bug (Label 2). Reviews discussing the app's user interface with no mention of privacy should be labeled as Not Privacy-Related (Label 0).
Additional Notes Focus on the main context if the review contains both privacy-related aspects and other topics. Use your best judgment if the review is ambiguous, and aim for consistency in labeling.
